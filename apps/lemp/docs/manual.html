LEMP Stack — คู่มือการใช้งาน

Dev/Base Stack: Nginx + PHP-FPM + MariaDB สำหรับพัฒนาเว็บ PHP

ภาพรวม

LEMP Stack คือชุดซอฟต์แวร์สำหรับพัฒนาเว็บ PHP แบบสำเร็จรูป ประกอบด้วย Nginx (เว็บเซิร์ฟเวอร์), PHP-FPM (ตัวประมวลผล PHP), และ MariaDB (ฐานข้อมูล) — พร้อมใช้งานทันทีหลังจากบูตเครื่องครั้งแรก

Image นี้เป็น Dev/Base Stack สำหรับนักพัฒนาที่ต้องการ environment PHP + DB ที่ไม่ต้องติดตั้งเอง ไม่มี app wizard หรือหน้า welcome — แค่เปิด browser แล้วเริ่มเขียนโค้ด PHP ได้เลย

Stack Components

  • Nginx 1.30.3 — เว็บเซิร์ฟเวอร์, reverse proxy ไปยัง PHP-FPM
  • PHP 8.3.32 (FPM) — ตัวประมวลผล PHP พร้อม extensions: pdo_mysql, mysqli, mbstring, gd, zip, intl
  • MariaDB 11.4.12 (LTS) — ฐานข้อมูล, รองรับจนถึง ~2029

คุณสมบัติเด่น

  • Offline-safe — images ถูก build ไว้ใน golden image แล้ว ไม่ต้องดึงจาก internet ตอนบูต
  • Idempotent bootstrap — รันซ้ำกี่ครั้งก็ไม่พัง, ใช้ .env เดิมถ้ามี
  • Security hardening — server_tokens off, expose_php = Off, DB/PHP-FPM ไม่ expose port
  • Dev tools ใน container — bash, nano, vim-tiny พร้อมในทุก container
  • Helper commands — คำสั่งสั้น จำง่าย: lemp-status, lemp-logs, lemp-restart, lemp-shell, lemp-db

HTTP / HTTPS Profiles

LEMP Stack มี 2 profiles ให้เลือกใช้:

HTTP Profile (ค่าเริ่มต้น)

เปิด port 80 เท่านั้น — ใช้ตอน first boot โดยอัตโนมัติ

docker compose --profile http up -d

HTTPS Profile

เปิด port 80 + 443 — ต้องมี certificate files ก่อน

1

ชี้ DNS ไปที่ VM IP

2

วาง certificate files ที่ /opt/lemp/certs/:

/opt/lemp/certs/fullchain.pem
/opt/lemp/certs/privkey.pem
3

ตั้ง permission:

chmod 644 /opt/lemp/certs/fullchain.pem
chmod 600 /opt/lemp/certs/privkey.pem
4

เปลี่ยน profile เป็น HTTPS:

cd /opt/lemp
docker compose --profile http down
docker compose --profile https up -d
แนะนำ: ใช้ Let's Encrypt + certbot สำหรับ certificate ฟรี หรือใช้ self-signed cert สำหรับ dev

Backup & Restore

Backup Database

cd /opt/lemp
docker compose exec -T db mariadb-dump -u root -p"$(grep 'Root DB Password' /root/lemp-credentials.txt | awk -F': ' '{print $2}')" lempdb > /root/lempdb-backup-$(date +%Y%m%d).sql

Backup Files/Config

tar czf /root/lemp-config-backup-$(date +%Y%m%d).tar.gz /opt/lemp/

Restore Database

cd /opt/lemp
cat /root/lempdb-backup-20260712.sql | docker compose exec -T db mariadb -u root -p"$(grep 'Root DB Password' /root/lemp-credentials.txt | awk -F': ' '{print $2}')" lempdb

Restore Files/Config

cd /
tar xzf /root/lemp-config-backup-20260712.tar.gz
cd /opt/lemp && docker compose --profile http up -d
สำคัญ: Backup ก่อนอัปเกรดหรือรีเซ็ตรหัสผ่านเสมอ ข้อมูลใน Docker volume จะหายเมื่อรัน docker compose down -v

การอัปเกรด

1

Backup ก่อนอัปเกรดเสมอ — ดูหัวข้อ Backup & Restore

2

Pull Docker images ใหม่:

cd /opt/lemp
docker compose --profile http pull
3

Restart services ด้วย image ใหม่:

cd /opt/lemp
docker compose --profile http up -d
4

ตรวจสอบว่าเวอร์ชันอัปเกรดแล้ว:

lemp-shell -c 'php -v | head -1'
docker compose exec -T nginx nginx -v 2>&1
docker compose exec -T db mariadb --version
ข้อควรระวัง: PHP 8.3 → 8.4 อาจมี breaking changes ตรวจสอบ compatibility ของโค้ดคุณก่อนอัปเกรด MariaDB minor version (11.4.x) ปลอดภัยกว่า major version (11.4 → 11.5)

แก้ไขปัญหา

Nginx 403 Forbidden

สาเหตุ: ยังไม่มี index.php ใน /var/www/html/

แก้: สร้างไฟล์ PHP ผ่าน lemp-shell:

lemp-shell
echo " /var/www/html/index.php
exit

502 Bad Gateway

สาเหตุ: PHP-FPM container ไม่ทำงานหรือยังไม่พร้อม

แก้: ตรวจสอบสถานะและ restart:

lemp-status
lemp-restart php-fpm

MariaDB connection refused

สาเหตุ: MariaDB ยังไม่พร้อม หรือ healthcheck ยังไม่ผ่าน

แก้: รอสักครู่แล้วลองใหม่ หรือตรวจสอบ log:

lemp-logs db

Container ไม่ restart หลัง docker kill

สาเหตุ: docker kill ไม่ trigger unless-stopped restart policy (โดย design ของ Docker)

แก้: ใช้ lemp-restart แทน:

lemp-restart

Bootstrap สร้าง secret ใหม่ทุกครั้ง

สาเหตุ: ไฟล์ /opt/lemp/.env ถูกลบ

แก้: ปกติ — bootstrap จะ reuse .env ถ้ามี ถ้าต้องการ reset จริงๆ ให้ลบ .env + credentials + log แล้วรัน bootstrap ใหม่

VM IP ไม่ตรงใน credentials

สาเหตุ: credentials ถูกสร้างตอน first boot — IP เปลี่ยนหลังจากนั้น

แก้: ไม่มีผลต่อการทำงานของ stack แค่ credentials file แสดง IP เก่า ถ้าต้องการอัปเดต ให้ลบ .env + credentials + log แล้วรัน bootstrap ใหม่
