n8n — User Manual

Workflow Automation image with n8n, PostgreSQL 16, and Nginx reverse proxy.

Overview

This image runs n8n automatically on Ubuntu 26.04 using Docker Compose. On first boot, the environment creates secure database credentials and a unique encryption key, keeping your workflow secrets completely safe.

Stack

  • n8n — Workflow automation engine
  • PostgreSQL 16 — Stable database for robust performance and scaling
  • Nginx — Reverse proxy to handle HTTPS termination for secure webhooks

Access

By default, n8n runs behind an Nginx reverse proxy on port 80. You will need to create your owner account on first access.

1

Open http://<VM-IP>/ in your browser.

Note: The n8n service itself listens on port 5678 internally (localhost only). External traffic goes through Nginx on port 80. Do not open port 5678 in your security group.
2

Follow the on-screen instructions to create your n8n owner account.

3

You can now start building workflows and connecting apps!

Credentials

The system generated a random database password and N8N_ENCRYPTION_KEY on the first boot.

View the credentials file

cat /root/n8n-credentials.txt
CRITICAL: Please backup the N8N_ENCRYPTION_KEY found in this file immediately. If you lose this key, you will permanently lose access to all saved app credentials inside n8n.

HTTPS Setup

Webhooks from services like Stripe or Slack require a secure HTTPS connection. This image makes it easy to switch to HTTPS mode.

1

Point your domain (e.g. n8n.yourdomain.com) to this VM's IP.

2

Place your SSL certificate and private key in the certs directory:

/opt/n8n/certs/fullchain.pem
/opt/n8n/certs/privkey.pem
3

Edit the /opt/n8n/.env file to reflect your domain:

N8N_HOST=n8n.yourdomain.com
N8N_PROTOCOL=https
WEBHOOK_URL=https://n8n.yourdomain.com/
N8N_SECURE_COOKIE=true
4

Activate HTTPS nginx config and restart:

cd /opt/n8n
cp nginx/n8n-https.conf nginx/n8n.conf
docker compose down
docker compose up -d
Tip: Use the n8n-https-enable helper to automate all steps above. It updates .env, copies the right nginx config, and restarts the stack for you.

Helper Commands

The image installs these helpers at /usr/local/bin/ for daily management tasks. You can run them from any directory.

General management

n8n-logsShow recent logs (last 50 lines). Pass service name for specific logs: n8n-logs nginx

HTTPS management

CommandPurpose

n8n-https-disableDisable HTTPS, revert to HTTP mode. Cert files are preserved
n8n-cert-statusShow SSL certificate status, expiry, and current mode (HTTP/HTTPS)

Tip: All helpers operate from the /opt/n8n directory automatically, so you can run them from any working directory.

Backup & Restore

Warning: Do NOT run docker compose down -v. It will permanently delete PostgreSQL and n8n data volumes.

Back up the database

cd /opt/n8n
source .env
docker compose exec postgres pg_dump -U "$POSTGRES_USER" "$POSTGRES_DB" > n8n-db-backup-$(date +%Y%m%d).sql

Back up config and files

cd /opt/n8n
tar czf n8n-config-backup-$(date +%Y%m%d).tar.gz .env docker-compose.yml nginx/

Restore the database

cd /opt/n8n
source .env
cat n8n-db-backup-YYYYMMDD.sql | docker compose exec -T postgres psql -U "$POSTGRES_USER" "$POSTGRES_DB"

Restore config

cd /opt/n8n
tar xzf n8n-config-backup-YYYYMMDD.tar.gz
docker compose restart
Note: Always back up your N8N_ENCRYPTION_KEY separately — see the Credentials section. Without this key, stored n8n credentials become undecryptable.

Upgrade & Rollback

The image provides two helper commands for safe upgrading and rollback.

Upgrade

1

Run the upgrade helper:

n8n-upgrade

This saves the current n8n image tag to /opt/n8n/.previous-image, pulls the newer tag from docker-compose.yml, and restarts n8n.

2

Verify the stack comes back cleanly:

n8n-status

Rollback

If the upgrade causes trouble, restore the previous version:

n8n-rollback

This reads the saved tag from /opt/n8n/.previous-image, updates docker-compose.yml, pulls the old image, and restarts.

Warning: Rollback only works if n8n-upgrade was run previously. Always keep a separate database backup before upgrading — see Backup & Restore.

Service Management

Use Docker Compose to manage your n8n server from the /opt/n8n directory.

Check status

cd /opt/n8n
docker compose ps

Check logs

cd /opt/n8n
docker compose logs -f

Restart all services

cd /opt/n8n
docker compose restart

Stop all services

cd /opt/n8n
docker compose down

Start all services

cd /opt/n8n
docker compose up -d
Note: The nginx reverse proxy runs in all modes (HTTP and HTTPS). There are no Docker Compose profiles — mode switching is done via nginx config files and .env settings.
