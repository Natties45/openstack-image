Nextcloud — คู่มือการใช้งาน

ระบบแชร์ไฟล์และทำงานร่วมกันแบบ Private Cloud — พร้อมใช้งานทันทีหลังจากสร้าง VM

ภาพรวม

Nextcloud คือแพลตฟอร์มแชร์ไฟล์และทำงานร่วมกันแบบ open-source ติดตั้งบน VM ของคุณเอง ข้อมูลทั้งหมดอยู่บนเซิร์ฟเวอร์ของคุณ ไม่ได้ส่งไปยังบริษัทอื่น

Image นี้ถูกออกแบบให้สร้าง VM แล้วเปิด browser เข้าใช้งานได้ทันที — ไม่ต้องตั้งค่าอะไรเพิ่ม

Stack ที่ใช้

  • Nextcloud 30.0 — ตัวแอปพลิเคชันหลัก (PHP 8.4 + Apache)
  • PostgreSQL 16.9 — ฐานข้อมูล
  • Redis 7.4 — ระบบ cache และ file locking (เพิ่มความเร็ว)
  • Nginx 1.27 — reverse proxy (จัดการ SSL และเชื่อมต่อภายนอก)
  • Docker Compose — จัดการ containers ทั้ง 4 ตัว

คุณสมบัติเด่น

  • สร้าง VM → เปิด browser → ใช้งานได้เลย — ไม่ต้องตั้งค่า
  • IP เปลี่ยนอัตโนมัติ — ทดสอบจริงหลัง shutoff/change IP แล้วเข้าใช้งานได้ทันที
  • Redis cache + file locking — ทำงานเร็วขึ้น
  • System cron รันทุก 5 นาที — งานพื้นหลังไม่ค้าง
  • Upgrade/Rollback ด้วยคำสั่งเดียว — ไม่เสียข้อมูล

Security Group (Firewall)

เปิด port ต่อไปนี้ที่ OpenStack Security Group เพื่อให้เข้าใช้งาน Nextcloud ได้:

PortProtocolจำเป็น?用途
22TCP✅ จำเป็นSSH — สำหรับผู้ดูแลระบบเข้า VM
80TCP✅ จำเป็นHTTP — เปิดเว็บ Nextcloud (ต้องมีอย่างน้อย 1 port)
443TCP⚠️ แล้วแต่HTTPS — ถ้าต้องการเปิด SSL (เพิ่มเติมจาก port 80)
หมายเหตุ: container ports (PostgreSQL 5432, Redis 6379, Apache 80) อยู่ใน Docker internal network ไม่ต้องเปิดใน Security Group

ขั้นตอนเข้าใช้งานครั้งแรก

1

เปิด browser ไปที่ http://<VM-IP>/

2

จะเห็นหน้า Login — ใช้ username admin และ password จากไฟล์ /root/nextcloud-credentials.txt (ดูวิธีในหัวข้อ รหัสผ่าน)

3

เมื่อ login สำเร็จ จะเข้าสู่หน้า Dashboard — พร้อมใช้งาน

แนะนำ: เปลี่ยนรหัสผ่าน admin หลัง login ครั้งแรก ไปที่ Settings → Security → Devices & sessions → Password
ตรวจสอบจริง: image นี้ทดสอบแล้วทั้ง boot ใหม่จาก image และกรณี shutoff + เปลี่ยน public IP โดย bootstrap จะเติม trusted_domains ใหม่อัตโนมัติ ทำให้เปิด http://<VM-IP>/ ได้ทันทีโดยไม่ต้องแก้ config เอง

รหัสผ่าน & Credentials

ดูรหัสผ่าน

รหัสผ่านทั้งหมดถูกสุ่มตอนสร้าง VM ครั้งแรก และเก็บไว้ในไฟล์:

cat /root/nextcloud-credentials.txt

ไฟล์นี้ประกอบด้วย:

  • Nextcloud Admin — username: admin, password สำหรับ login เว็บ
  • PostgreSQL Database — สำหรับเชื่อมต่อฐานข้อมูลโดยตรง
  • Redis — password สำหรับ Redis cache

รีเซ็ตรหัสผ่าน admin

ใช้คำสั่ง nc-occ เพื่อรีเซ็ตรหัสผ่าน:

nc-occ user:resetpassword admin

ระบบจะถามรหัสผ่านใหม่ — ใส่รหัสผ่านที่ต้องการ

รีเซ็ตรหัสผ่าน PostgreSQL

cd /opt/nextcloud
docker compose exec db psql -U nextcloud -c "ALTER USER nextcloud PASSWORD 'รหัสผ่านใหม่';"

จากนั้นอัปเดตไฟล์ /opt/nextcloud/.env ด้วยรหัสผ่านใหม่

คำเตือน: ไฟล์ /opt/nextcloud/.env และ /root/nextcloud-credentials.txt มีรหัสผ่านทั้งหมด — ห้ามแชร์ ห้าม dump ลง chat หรือ repo

คำสั่ง Helper

Image นี้มีคำสั่งลัดสำหรับงานที่ใช้บ่อย — ใช้ได้ทุกที่ ไม่ต้อง cd เข้า /opt/nextcloud

รายการคำสั่ง

คำสั่งทำอะไรตัวอย่าง

ตัวอย่างการใช้งาน

# ตรวจสอบสถานะ Nextcloud
nc-occ status

nc-occ user:list

# ดู trusted domains
nc-occ config:system:get trusted_domains

# ตรวจสอบ container
nc-status

# ดู log ของ Nextcloud
nc-logs nextcloud

# Restart ทั้ง stack

nc-upgrade

หมายเหตุ: nc-restart ใช้ --profile http เสมอ ถ้าคุณเปิด HTTPS อยู่ ให้ใช้ docker compose --profile https restart แทน

การสลับ HTTP/HTTPS

Nextcloud image นี้ใช้ Docker Compose profiles เพื่อแยกโหมด HTTP และ HTTPS — คุณสามารถสลับไปมาได้โดยไม่ต้องแก้ config

Profile ที่มี

ProfilePortเมื่อไหร่
http (default)80ใช้งานครั้งแรก, ทดสอบ, หรือยังไม่มี domain
https80 + 443มี domain + certificate แล้ว

เปิดโหมด HTTP (default)

cd /opt/nextcloud
docker compose --profile http up -d

เปิดโหมด HTTPS

ดูขั้นตอนในหัวข้อ การเปิด HTTPS

สลับจาก HTTP → HTTPS

1

วาง certificate ที่ /opt/nextcloud/certs/

2

Stop Nginx HTTP:

cd /opt/nextcloud
docker compose --profile http stop nginx
3

Start HTTPS profile:

docker compose --profile https up -d

สลับจาก HTTPS → HTTP

1

Stop HTTPS Nginx:

cd /opt/nextcloud
docker compose --profile https stop nginx-https
2

Start HTTP profile:

docker compose --profile http up -d
สำคัญ: port 80 ถูกใช้โดยทั้ง Nginx HTTP และ Nginx HTTPS — ไม่สามารถรันทั้ง 2 profile พร้อมกันได้ ต้อง stop profile เก่าก่อน start profile ใหม่เสมอ

การเปิด HTTPS

1

ชี้ DNS ไปที่ VM floating IP

2

วาง certificate files ที่ /opt/nextcloud/certs/:

/opt/nextcloud/certs/fullchain.pem
/opt/nextcloud/certs/privkey.pem
3

ตั้ง permission:

chmod 644 /opt/nextcloud/certs/fullchain.pem
chmod 600 /opt/nextcloud/certs/privkey.pem
4

เปิด HTTPS profile (ดูหัวข้อ การสลับ HTTP/HTTPS):

cd /opt/nextcloud
docker compose --profile https up -d
5

ตรวจสอบ:

curl -sI https://yourdomain.com | head -1
# ต้องได้: HTTP/2 200

ขอ certificate ฟรีด้วย Let's Encrypt

# ติดตั้ง certbot
apt install certbot

# ขอ certificate (ต้องมี DNS ชี้มาที่ IP แล้ว)
certbot certonly --manual -d yourdomain.com --preferred-challenges dns

# จะได้ไฟล์:
#   /etc/letsencrypt/live/yourdomain.com/fullchain.pem
#   /etc/letsencrypt/live/yourdomain.com/privkey.pem

# copy ไปวาง
cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem /opt/nextcloud/certs/
cp /etc/letsencrypt/live/yourdomain.com/privkey.pem /opt/nextcloud/certs/
แนะนำ: ใช้ Let's Encrypt สำหรับ certificate ฟรี อายุ 90 วัน ต้อง renew ทุก 3 เดือน

Backup & Restore

สำคัญ: คำสั่ง backup ด้านล่างนี้เป็นการ backup ข้อมูลทั้งหมด (DB + files) — ควรทำก่อน upgrade หรือก่อนการเปลี่ยนแปลงสำคัญ

Backup Database

cd /opt/nextcloud
docker compose exec db pg_dump -U nextcloud nextcloud > nc-db-backup-$(date +%Y%m%d).sql

Backup Files + Config

# Backup runtime data ทั้งหมด
tar czf nc-data-backup-$(date +%Y%m%d).tar.gz -C /var/lib nextcloud

# Backup config/control files
tar czf nc-config-backup-$(date +%Y%m%d).tar.gz -C /opt nextcloud

Restore Database

cd /opt/nextcloud
docker compose exec -T db psql -U nextcloud nextcloud < nc-db-backup-YYYYMMDD.sql

Restore Files

tar xzf nc-data-backup-YYYYMMDD.tar.gz -C /var/lib
docker compose --profile http restart
สำคัญ: การ restore ควรทำในขณะที่ Nextcloud อยู่ใน maintenance mode เสมอ: 
nc-occ maintenance:mode --on
# ... ทำ restore ...
nc-occ maintenance:mode --off

การอัปเกรด

Image นี้มีคำสั่ง nc-upgrade และ nc-rollback สำหรับอัปเกรดและย้อนกลับ — ไม่ต้อง backup ข้อมูล เพราะ rollback แค่เปลี่ยน image tag ไม่ได้แตะ data volume

อัปเกรด (แนะนำ)

1

รันคำสั่งอัปเกรด:

nc-upgrade

คำสั่งนี้จะ:

  • บันทึก image tag ปัจจุบันไปที่ /opt/nextcloud/.previous-image
  • เปิด maintenance mode
  • pull image ใหม่
  • restart containers
  • รอให้ Nextcloud พร้อม
  • ปิด maintenance mode

ย้อนกลับ (Rollback)

ถ้าอัปเกรดแล้วมีปัญหา:

1

รันคำสั่ง rollback:

nc-rollback

คำสั่งนี้จะ:

  • อ่าน image tag เดิมจาก /opt/nextcloud/.previous-image
  • เปิด maintenance mode
  • pull image เดิม
  • แก้ docker-compose.yml กลับเป็น tag เดิม
  • restart containers
  • ปิด maintenance mode

อัปเกรดด้วยตนเอง (Manual)

cd /opt/nextcloud

# เปิด maintenance mode
nc-occ maintenance:mode --on

# pull image ใหม่
docker compose pull nextcloud

# restart
docker compose up -d

# ปิด maintenance mode
nc-occ maintenance:mode --off
ข้อควรระวัง:
  • Nextcloud ข้าม major version ไม่ได้ — ต้อง upgrade ทีละ version (เช่น 29→30→31)
  • ตรวจสอบ compatibility ก่อน upgrade ทุกครั้งที่ docs.nextcloud.com
  • ถ้า rollback ไม่มีไฟล์ .previous-image — rollback ไม่ได้ (กรณีนี้ต้อง restore จาก backup)
