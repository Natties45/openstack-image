Grafana+Prometheus · v1

Grafana+Prometheus — คู่มือการใช้งาน

ระบบ Monitoring สำเร็จรูป — ตรวจสอบสถานะ VM, เว็บไซต์, และบริการต่างๆ ผ่าน Grafana Dashboard

ภาพรวม

VM เครื่องนี้มาพร้อมระบบ Monitoring แบบครบวงจร ทำงานอัตโนมัติตั้งแต่เปิดเครื่องครั้งแรก ประกอบด้วย:

Componentบทบาท
Grafanaหน้าจอ Dashboard — แสดงกราฟ, สถานะ, และ alert
Prometheusเก็บบันทึก metrics — ดึงข้อมูลจาก target ต่างๆ ตามเวลา
Alertmanagerส่งแจ้งเตือน — เมื่อเกิดเหตุการณ์ตามเงื่อนไขที่ตั้งไว้
node_exporterเก็บ metrics ของ VM เครื่องนี้ — CPU, RAM, disk, network
blackbox_exporterตรวจสอบ HTTP, HTTPS, TCP, ping จากภายนอก
NginxReverse proxy — รับ request จาก browser ส่งต่อให้ Grafana
cAdvisorตัวเลือกเสริม — เก็บ metrics ของ Docker containers

การทำงาน

  • เปิด VM → ระบบสร้าง password อัตโนมัติ → start ทุก service → พร้อมใช้งานทันที
  • VM เครื่องนี้ monitor ตัวเองผ่าน node_exporter อัตโนมัติ
  • เพิ่มเครื่องอื่น, เว็บไซต์, หรือพอร์ตที่ต้องการ monitor ได้ด้วยคำสั่ง monitoring-add-*
  • ตั้งค่า alert แจ้งเตือนผ่าน email, LINE, Slack ฯลฯ ได้
  • Prometheus เก็บ metrics ย้อนหลัง 30 วัน (ปรับได้)

รหัสผ่าน & Credentials

ดูรหัสผ่าน

รหัสผ่านถูกสุ่มตอนเปิด VM ครั้งแรก — ใช้คำสั่งนี้ดู:

sudo monitoring-info

แสดงผล:

Grafana URL:
  http://10.0.0.5/

Login:
  Username: admin
  Password: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

รีเซ็ตรหัสผ่าน

sudo monitoring-reset-grafana-password

คำสั่งนี้จะ:

  • สุ่ม password ใหม่
  • อัปเดต password ใน Grafana โดยตรง
  • อัปเดตไฟล์ /opt/monitoring/.env
  • เขียนไฟล์ /root/README-grafana-prometheus-image.txt ใหม่
คำเตือน: ไฟล์ /opt/monitoring/.env และ /root/README-grafana-prometheus-image.txt มี password — ตั้ง permission 600 ห้ามให้คนอื่นอ่าน

ตำแหน่งไฟล์ที่เก็บ password

ไฟล์เนื้อหาPermission
/opt/monitoring/.envGRAFANA_ADMIN_PASSWORD, GRAFANA_SECRET_KEY600 (root เท่านั้น)
/root/README-grafana-prometheus-image.txtPassword + ผังไฟล์ (human-readable)600 (root เท่านั้น)

Backup & Restore

สิ่งที่ต้อง Backup

ข้อมูลVolumeวิธี Backup
Grafana — dashboards, users, settingsgrafana_dataCopy volume หรือ export dashboard JSON
Prometheus — metrics ย้อนหลังprometheus_dataCopy volume (ข้อมูลใหญ่ — ใช้ rsync)
Alertmanager — alert state, silencesalertmanager_dataCopy volume
Config files/opt/monitoring/tar ไฟล์ config ทั้งหมด

Backup Volumes

cd /opt/monitoring
docker compose down

# Backup volumes
docker run --rm -v grafana_data:/data -v /backup:/backup alpine tar czf /backup/grafana-data.tar.gz -C /data .
docker run --rm -v prometheus_data:/data -v /backup:/backup alpine tar czf /backup/prometheus-data.tar.gz -C /data .
docker run --rm -v alertmanager_data:/data -v /backup:/backup alpine tar czf /backup/alertmanager-data.tar.gz -C /data .

docker compose up -d

Backup Config

sudo tar czf /backup/monitoring-config.tar.gz -C /opt monitoring

Restore

cd /opt/monitoring
docker compose down

# Restore volumes
docker run --rm -v grafana_data:/data -v /backup:/backup alpine tar xzf /backup/grafana-data.tar.gz -C /data
docker run --rm -v prometheus_data:/data -v /backup:/backup alpine tar xzf /backup/prometheus-data.tar.gz -C /data
docker run --rm -v alertmanager_data:/data -v /backup:/backup alpine tar xzf /backup/alertmanager-data.tar.gz -C /data

# Restore config
sudo tar xzf /backup/monitoring-config.tar.gz -C /

docker compose up -d
สำคัญ: Prometheus data volume อาจมีขนาดใหญ่ (ขึ้นกับจำนวน target และ retention period) — ตรวจสอบพื้นที่ disk ก่อน backup เสมอ

การเพิ่ม Target สำหรับ Monitoring

ใช้คำสั่ง monitoring-add-* เพิ่ม target ได้ 4 ประเภท — คำสั่งจะเติม target ลงไฟล์ที่ถูกต้องและ reload Prometheus ให้อัตโนมัติ

เพิ่ม HTTP/HTTPS Target

ตรวจสอบว่าเว็บไซต์ตอบ HTTP 2xx หรือไม่

sudo monitoring-add-http https://example.com website-prod

เงื่อนไข: URL ต้องขึ้นต้นด้วย http:// หรือ https://

target จะถูกเพิ่มใน /opt/monitoring/prometheus/targets/http.yml

เพิ่ม Node Target (Linux VM)

ตรวจสอบ metrics ของเครื่องอื่นผ่าน node_exporter (ต้องติดตั้ง node_exporter ที่เครื่องปลายทางก่อน)

sudo monitoring-add-node 10.0.0.12:9100 web-01

เงื่อนไข: เครื่องปลายทางต้องเปิด port 9100 และติดตั้ง node_exporter

target จะถูกเพิ่มใน /opt/monitoring/prometheus/targets/nodes.yml

เพิ่ม TCP Port Target

ตรวจสอบว่า TCP port เปิดอยู่หรือไม่

sudo monitoring-add-tcp 10.0.0.20:5432 postgres-01

target จะถูกเพิ่มใน /opt/monitoring/prometheus/targets/tcp.yml

เพิ่ม Ping Target

ตรวจสอบว่า IP ตอบ ICMP ping หรือไม่

sudo monitoring-add-ping 10.0.0.30 router-01

target จะถูกเพิ่มใน /opt/monitoring/prometheus/targets/ping.yml

ดู Targets ปัจจุบัน

sudo monitoring-list-targets

ลบ Target

แก้ไขไฟล์ target โดยตรง:

sudo nano /opt/monitoring/prometheus/targets/http.yml

แล้ว reload:

sudo monitoring-reload
การเปิด cAdvisor (Docker container metrics): cAdvisor ใช้ Docker Compose profile — ต้องเปิดแยก: 
cd /opt/monitoring
docker compose --profile cadvisor up -d
แล้วเพิ่ม target ด้วย sudo monitoring-add-node <host>:8080 cadvisor (ต้องมี cAdvisor ที่ target)

การตั้งค่า Alert

Alert Rules — เงื่อนไขการแจ้งเตือน

แก้ไขไฟล์ Prometheus alert rules:

sudo nano /opt/monitoring/prometheus/rules/alerts.yml

ตัวอย่าง alert rule — แจ้งเมื่อ CPU สูงเกิน 90% นาน 5 นาที:

groups:
  - name: node_alerts
    rules:
      - alert: HighCPUUsage
        expr: 100 - (avg(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 90
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "CPU usage > 90% on {{ $labels.instance }}"

หลังแก้ไข:

sudo monitoring-reload

Alertmanager — ช่องทางการส่ง Alert

แก้ไขไฟล์ Alertmanager config:

sudo nano /opt/monitoring/alertmanager/alertmanager.yml

ตัวอย่าง — ส่ง alert ผ่าน email และ Slack:

global:
  smtp_smarthost: 'smtp.gmail.com:587'
  smtp_from: 'alert@example.com'
  smtp_auth_username: 'alert@example.com'
  smtp_auth_password: 'app-password'

route:
  receiver: 'default'

receivers:
  - name: 'default'
    email_configs:
      - to: 'admin@example.com'
    slack_configs:
      - api_url: 'https://hooks.slack.com/services/...'
        channel: '#alerts'

หลังแก้ไข:

cd /opt/monitoring
docker compose restart alertmanager
Alertmanager รองรับ receivers: email, Slack, LINE (ผ่าน LINE Notify), PagerDuty, OpsGenie, Webhook, WeChat, Telegram และอื่นๆ

ดูสถานะ Alert

  • Prometheus Alerts: เปิด Grafana → ไปที่เมนู Alerting → Alert rules
  • Alertmanager: http://<VM-IP>:9093 (เฉพาะภายใน VM) หรือดูผ่าน docker logs alertmanager
